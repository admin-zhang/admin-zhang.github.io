首先，我们应该知道 == 对于基本类型和引用类型所起到的作用是不同的：

  • 对基本类型来说：比较的就是值是否相同；
  • 对引用类型来说：比较的就是地址值是否相同，也就是对象的内存地址

在上述代码中， String 和 Integer 均属于引用类型，故 == 比较的就是对象的内存地址是否相等；所以对于上述代码执行结果我们可以做如下分析：

  1. String

    ​ 对于 String 类来说， 在 Java 中，使用双引号创建字符串时，JVM 首先会去字符串常量池中查找是否有相同值的字符串。如果有，则返回对该字符串的引用，否则会在字符串常量池中创建一个新的字符串对象。因此，对于字符串字面值 "aaa" ，JVM 会检查字符串池是否已经存在一个具有相同值的字符串。

    ​ 所以在这种情况下 String a = "aaa"; 这行代码在执行时，JVM 会在字符串池中创建一个对象，该对象值为 "aaa" ，并将 a 设置为引用该对象。

    ​ 而对于 String b = new String("aaa"); 来说，其会创建一个新的字符串对象，又因为其使用了 new 关键字，此时会显式地在堆内存中创建，并且它的值也是 aaa ，但是不同的是，这个对象并没有存储在字符串常量池中，而是存储在堆中，因此变量 b 引用的是堆中的这个新创建的字符串对象。

    ​ 所以在上述代码中， a=="aaa" 的结果为 true ， b=="aaa" 的结果为 false 。

  2. Integer

    ​ 我们都知道 Integer 属于包装类，在 Java 中基本数据类型和其对应的包装类型之间可以进行自动装箱和自动拆箱。自动装箱是将基本数据类型转换为对应的包装类对象，而自动拆箱是将包装类对象转换为对应的基本数据类型值。

    ​ 在上述例子中，由于 Integer 是包装类，其在执行过程中会发生自动装箱的过程，而在这个过程中，又因为其对应的基本类型是 int , 其取值范围在 -128 到 127 之间，所以在创建新的 Integer 对象时，若处于这个范围内 ，那么在自动装箱时会查找缓存的值，若超出该范围，则会创建一个新的对象，所以对于上述代码中 c == d 的执行结果为 true , e == f 的执行结果为 false 。
